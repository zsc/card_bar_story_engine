# 场景：家门口（清晨启动）

**氛围**：清晨的光还没完全醒，车里先醒了。屏幕亮起，语音链路稳定在车端。

**可见角色**：
- 座舱助手（CDI Cockpit）
- 驾驶助手（CDI Driving）

**可体现的 CDI 关键点**：
- 分层：一个负责安全闭环，一个负责对话与决策
- "先结论后理由"的简报
- 交互模式选择（驾驶优先/生活优先/最少干预/离线优先）

**对话参考（可直接用或改写）**：
【座舱助手】"我先确认一件事：你今天赶时间吗？如果赶，我只说关键；如果不赶，我可以把路线、日程和风险讲得更完整。"  
【你】"8:50 早会，别迟到。"  
【座舱助手】"收到。三句话版本：  
一，你 8:50 早会，建议 8:35 前进停车场；  
二，常用路线早高峰拥堵概率高，我给你两个路线选项；  
三，今天你可能会接到工作电话，我建议默认'停车后处理'。"  
【驾驶助手】"安全带未系。"  
【你】（伸手去扣）"好了。"  
【驾驶助手】"确认。盲区监测在线。"  
【座舱助手】"你要现在选交互模式吗？  
- 驾驶优先：我只在必要时插话  
- 生活优先：我会主动帮你做提醒与整理  
- 最少干预：你问我才答  
- 离线优先：我会先下载本地包，弱网时也能用"

**风险**：
- 开始阶段若同时处理太多事情，会提高负荷，降低注意力

**推荐行动**：
- 让座舱助手做"通勤简报"并选交互模式
- 直接出发，把决策压缩成"路线二选一"

**本回合可选行动**（必须给选择）：
- A. "三句话通勤简报 + 选一个交互模式"（偏稳、提升信任/体验点）  
- B. "直接导航出发，别说太多"（降低负荷，但信息少）  
- C. "我先把手机的消息处理一下再走"（可能增加负荷/降低注意力）
